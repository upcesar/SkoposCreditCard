Constructor

Ink.UI.DatePicker

(
  • selector
  • [options]
  • [options.ofText='
)

Defined in Ink/UI/DatePicker/1/lib.js:39

Parameters:

  • selector String | DOMElement
  • [options] Object optional

    Options

    • [autoOpen=false] Boolean optional

      set to true to automatically open the datepicker.

    • [cleanText] String optional

      text to display on clean button. defaults to 'Limpar'

    • [closeText] String optional

      text to display on close button. defaults to 'Fechar'

    • [cssClass] String optional

      CSS class to be applied to the datepicker

    • [dateRange] String optional

      enforce limits to year, month and day for the Date, ex: '1990-08-25:2020-11'

    • [displayInSelect] Boolean optional

      whether to display the component in a select. defaults to false.

    • [dayField] String | DOMElement optional

      (if using options.displayInSelect) <select> field with days.

    • [monthField] String | DOMElement optional

      (if using options.displayInSelect) <select> field with months.

    • [yearField] String | DOMElement optional

      (if using options.displayInSelect) <select> field with years.

    • [format] String optional

      Date format string

    • [instance] String optional

      unique id for the datepicker

    • [month] Object optional

      Hash of month names. Defaults to portuguese month names. January is 1.

    • [nextLinkText] String optional

      text to display on the previous button. defaults to '«'

    • [onFocus=true] Boolean optional

      if the datepicker should open when the target element is focused

    • [onMonthSelected] Function optional

      callback function to execute when the month is selected

    • [onSetDate] Function optional

      callback to execute when set date

    • [onYearSelected] Function optional

      callback function to execute when the year is selected

    • [position] String optional

      position the datepicker. Accept right or bottom, default is right

    • [prevLinkText] String optional

      text to display on the previous button. defaults to '«'

    • [showClean] Boolean optional

      whether to display the clean button or not. defaults to true.

    • [showClose] Boolean optional

      whether to display the close button or not. defaults to true.

    • [shy=true] Boolean optional

      whether the datepicker starts automatically.

    • [startDate] String optional

      Date to define init month. Must be in yyyy-mm-dd format

    • [startWeekDay] Number optional

      day to use as first column on the calendar view. Defaults to Monday (1)

    • [validYearFn] Function optional

      callback function to execute when 'rendering' the month (in the month view)

    • [validMonthFn] Function optional

      callback function to execute when 'rendering' the month (in the month view)

    • [validDayFn] Function optional

      callback function to execute when 'rendering' the day (in the month view)

    • [nextValidDateFn] Function optional

      Find the next valid date, given the current Date. Necessary when the calendar has a lot of "holes", and not many dates are valid and they are separated by many months or years so it doesn't make sense to have the user click the "next" button too many times and not see any valid date.

    • [prevValidDateFn] Function optional

      See nextValidDateFn. Find the previous valid date.

    • [wDay] Object optional

      Hash of weekdays. Defaults to portuguese month names. Sunday is 0.

    • [yearRange] String optional

      enforce limits to year for the Date, ex: '1990:2020' (deprecated)

  • [options.ofText=' String

    of '] text to display between month and year. defaults to ' de '

Example:

<input type="text" id="dPicker" />
<script>
    Ink.requireModules(['Ink.Dom.Selector_1','Ink.UI.DatePicker_1'],function( Selector, DatePicker ){
        var datePickerElement = Ink.s('#dPicker');
        var datePickerObj = new DatePicker( datePickerElement );
    });
</script>
