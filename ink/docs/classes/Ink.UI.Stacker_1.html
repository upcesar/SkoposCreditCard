Ink.UI.Stacker_1 Class

Defined in: Ink/UI/Stacker/1/lib.js:12
Module: Ink.UI.Stacker_1

This module combines several stacks of items together, in smaller screen sizes.

The purpose is to have several stacks of items which may have different heights and as such cannot be used because of float: left quirks.

For example, when you have three different columns of information:

[col. A: 1] [col. B: 1] [col. C: 1]
[col. B: 2] [col. C: 2] [col. C: 2]

and the screen resizes and you need a layout of 2 columns, Stacker reorders the stacks so that you get:

[col. A: 1] [col. B: 1]
[col. C: 1] [col. A: 2]
[col. B: 2] [col. C: 2]

Note: If you just want to use a different amount of columns for your items in several viewports, but these items are guaranteed to have a fixed height, don't use this module. Use the small-*, medium-* and large-* classes instead.

Parameters:

  • [container] DOMElement | String optional

    Element which contains the stacks (identified by the options.column selector)

  • [options] Object optional
    • [column='.stacker-column'] String optional

      Select the columns inside the container

    • [item='.stacker-item'] String optional

      Select items in your stack

    • [customBreakPoints] Object optional

      options for each breakpoint name. Use this if you have more breakpoints than Ink by default (large, medium, small)

    • [customBreakpoints.(breakpoint)] Object optional
    • customBreakpoints.(breakpoint).max

      Maximum screen size as seen in your media query

    • customBreakpoints.(breakpoint).min

      Minimum screen size as seen in your media query

    • customBreakpoints.(breakpoint).cols

      Column count for this size.

    • [largeMax] Number optional

      Upper bound of large breakpoint

    • [largeMin=961] Number optional

      Lower bound of large breakpoint

    • [mediumMax=960] Number optional

      Upper bound of medium breakpoint

    • [mediumMin=651] Number optional

      Lower bound of medium breakpoint

    • [smallMax=650] Number optional

      Upper bound of small breakpoint

    • [smallMin] Number optional

      Lower bound of small breakpoint

    • [largeCols=3] Integer optional

      Number of columns in the large viewport

    • [mediumCols=2] Integer optional

      Number of columns in the medium viewport

    • [smallCols=1] Integer optional

      Number of columns in the small viewport

    • [isOrdered=true] Boolean optional

      When false, do not reorder stacks when combining them.

    • [onRunCallback] Function optional

      Called when instantiated

    • [onResizeCallback] Function optional

      Called when resized because the window resized.

    • [onAPIReloadCallback] Function optional

      Called when you ask to reload stack items from the DOM.

Example:

Html:

<div id="stacker-container">  <!-- Stacker element -->
    <div class="large-33 medium-50 small-100 stacker-column"> <!-- Column element ('.stacker-column' is the default selector) -->
        <div id="a" class="stacker-item">a</div> <!-- Item ('.stacker-item' is the default selector) -->
        <div id="d" class="stacker-item">d</div>
        <div id="g" class="stacker-item">g</div>
    </div>
    <div class="large-33 medium-50 small-100 hide-small stacker-column">
        <div id="b" class="stacker-item">b</div>
        <div id="e" class="stacker-item">e</div>
        <div id="h" class="stacker-item">h</div>
    </div>
    <div class="large-33 medium-50 small-100 hide-medium hide-small stacker-column">
        <div id="c" class="stacker-item">c</div>
        <div id="f" class="stacker-item">f</div>
        <div id="i" class="stacker-item">i</div>
    </div>
</div>

Javascript:

Ink.requireModules(['Ink.UI.Stacker_1'], function (Stacker) {
    var stacker = new Stacker('#stacker-container');
    // Keep the "stacker" variable around if you want to call addItem and reloadItems
});
