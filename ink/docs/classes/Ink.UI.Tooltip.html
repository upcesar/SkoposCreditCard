Defined in: Ink/UI/Tooltip/1/lib.js:8

Constructor

Ink.UI.Tooltip

(
  • target
  • [options]
)

Defined in Ink/UI/Tooltip/1/lib.js:8

Parameters:

  • target DOMElement | String

    Target element or selector of elements, to display the tooltips on.

  • [options] Object optional
    • [text=''] optional

      Text content for the tooltip.

    • [html=''] optional

      HTML for the tooltip. Same as above, but won't escape HTML.

    • [where='up'] optional

      Positioning for the tooltip. Options:

      • up/down/left/right
        Place above, below, to the left of, or to the right of, the target. Show an arrow.
      • mousemove
        Place the tooltip to the bottom and to the right of the mouse when it hovers the element, and follow the mouse as it moves.
      • mousefix
        Place the tooltip to the bottom and to the right of the mouse when it hovers the element, keep the tooltip there motionless.
    • [color=''] optional

      Color of the tooltip. Options are red, orange, blue, green and black. Default is white.

    • [fade=0.3] optional

      Fade time; Duration of the fade in/out effect.

    • [forever=0] optional

      Set to 1/true to prevent the tooltip from being erased when the mouse hovers away from the target

    • [timeout=0] optional

      Time for the tooltip to live. Useful together with [options.forever].

    • [delay] optional

      Time the tooltip waits until it is displayed. Useful to avoid getting the attention of the user unnecessarily

    • [template=null] optional

      Element or selector containing HTML to be cloned into the tooltips. Can be a hidden element, because CSS display is set to block.

    • [templatefield=null] optional

      Selector within the template element to choose where the text is inserted into the tooltip. Useful when a wrapper DIV is required.

    • [left,top=10] optional

      (Nitty-gritty) Spacing from the target to the tooltip, when where is mousemove or mousefix

    • [spacing=8] optional

      (Nitty-gritty) Spacing between the tooltip and the target element, when where is up, down, left, or right

Example:

<ul class="buttons">
    <li class="button" data-tip-text="Create a new document">New</li>
    <li class="button" data-tip-text="Exit the program">Quit</li>
    <li class="button" data-tip-text="Save the document you are working on">Save</li>
</ul>

[...]

<script>
    Ink.requireModules(['Ink.UI.Tooltip_1'], function (Tooltip) {
        new Tooltip('.button', {where: 'mousefix'});
    });
</script>

  • bbox
  • viewport
)

Defined in Ink/UI/Tooltip/1/lib.js:314

Get a value for "where" (left/right/up/down) which doesn't put the tooltip off the screen

Parameters:

  • where String

    "where" value which was given by the user and we might change

  • bbox BoundingBox

    A bounding box like what you get from getBoundingClientRect ({top, bottom, left, right}) with pixel positions from the top left corner of the viewport.

  • viewport BoundingBox

    Bounding box for the viewport. "top" and "left" are omitted because these coordinates are relative to the top-left corner of the viewport so they are zero.
